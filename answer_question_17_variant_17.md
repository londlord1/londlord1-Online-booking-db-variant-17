
17.Разница между поиском по индексированному столбцу и полнотекстовым поиском в MySQL заключается в механизме работы и типе решаемых задач. Поиск по индексированному столбцу использует B-Tree или Hash-индекс, обеспечивая мгновенное нахождение записей по точному совпадению или диапазону значений с логарифмической сложностью, однако становится бесполезным при использовании LIKE с шаблоном, начинающимся с символа подстановки, поскольку сервер вынужден выполнять полное сканирование таблицы. Полнотекстовый поиск строится на инвертированном индексе типа FULLTEXT, который разбивает текст на слова, отбрасывает стоп-слова и выполняет стемминг, позволяя искать по частичному совпадению слов в любом месте текста, по фразам с учётом порядка слов и ранжировать результаты по релевантности с помощью метрики, основанной на частоте встречаемости термина. Применительно к предметной области агентства недвижимости поиск по индексированному столбцу address идеально подходит для точного нахождения объекта по известному адресу, а индексы по price и area обеспечивают быструю фильтрацию по диапазону цен и площади. Полнотекстовый поиск был бы полезен при добавлении в таблицу properties текстового столбца description с описанием объектов, позволяя клиентам формулировать запросы на естественном языке, например искать квартиры с просторной кухней или видом на парк, и получать результаты, отсортированные по степени соответствия запросу. В текущей схеме полнотекстовый поиск не реализован, поскольку не предусмотрен столбец с развёрнутым описанием объектов, но при развитии системы он стал бы ценным дополнением к существующим индексам для повышения удобства пользователей.
