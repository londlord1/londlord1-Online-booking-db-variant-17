# Система онлайн-записи в агентство недвижимости (Вариант 17)

**Автор:** Молоков Александрович Алексей, группа 454
**Вариант:** 17. Система записи в агентство недвижимости на просмотр квартир. Риелторы ведут базу объектов. Клиент записывается на просмотр конкретного объекта в определённое время. Добавлена таблица избранных объявлений. Проверка того, что один объект не может быть показан двум разным клиентам в одно и то же время. Создайте запрос для вывода риелторов с наибольшим процентом успешных сделок по их записям.

## Описание предметной области
База данных автоматизирует процесс записи клиентов на просмотр объектов недвижимости. Клиенты могут выбирать объекты из каталога и записываться на удобное время. За каждым объектом закреплён риелтор, который проводит показы. Система позволяет добавлять объекты в избранное и отслеживать статус каждого просмотра (запланирован, проведён, сделка, отменён). Ключевое бизнес-правило: один объект не может быть показан двум разным клиентам в одно и то же время.

## Стек технологий
СУБД MySQL 8.0, SQL, Markdown для документации.

## Структура репозитория
В корне расположен файл README.md с описанием проекта. Папка sql содержит основной скрипт schema.sql для создания и наполнения базы данных, а также опциональный файл constraints_test.sql для проверки ограничений. Папка docs хранит отчёт о практической работе в формате Markdown (report.md) и подпапку images для скриншотов. Папка queries содержит отдельные SQL-файлы с тремя обязательными запросами: query1_join.sql, query2_group.sql, query3_top_realtors.sql. Папка diagrams предназначена для ER-диаграммы. Файл .gitignore исключает временные и системные файлы.

## Пошаговая инструкция по развёртыванию

### 1. Клонирование репозитория
git clone https://github.com/londlord1/online-booking-db-variant-17.git
cd online-booking-db-variant-17

### 2. Импорт базы данных через MySQL
Подключитесь к MySQL любым удобным способом (командная строка, Workbench, DBeaver) и выполните:
SOURCE /полный/путь/к/папке/sql/schema.sql;
Либо через командную строку:
mysql -u londlord1 -p < sql/schema.sql

### 3. Выполнение отдельных запросов
Откройте файлы из папки queries в вашем SQL-клиенте и выполните их по очереди для проверки работы базы данных.
